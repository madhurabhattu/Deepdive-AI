# DeepDive AI — Contributor Geographical Expansion Plan

This document details our approach to expanding the geographical diversity of our contributor network. By partnering with educational institutes and local software networks, we aim to build a decentralized team of maintainers.

---

## 1. Swecha Ecosystem Collaboration

DeepDive AI is hosted within the [Swecha GitLab](https://code.swecha.org) infrastructure. We leverage the Swecha ecosystem to drive contributor acquisition:
* **Infrastructure Support:** Using Swecha's shared computing resources to host mirrors, build runner nodes, and host preview environments.
* **Direct Outreach:** Sharing project news in Swecha's monthly newsletter and developer forums.
* **Swecha Summer Software Camps:** Positioning DeepDive AI as a core project for student internships, where students spend 4 weeks building open-source tools.

---

## 2. College Chapter Outreach

We actively engage with colleges and universities, establishing a structured pathway for students to transition from users to code contributors:

```
  ┌────────────────────────────────────────────────────────┐
  │ 1. Campus Presentation & Demonstration                 │
  └───────────────────────────┬────────────────────────────┘
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │ 2. Install-Fest: Configuring Ollama & Python Locally   │
  └───────────────────────────┬────────────────────────────┘
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │ 3. Localization Sprints: Writing translation dictionaries│
  └───────────────────────────┬────────────────────────────┘
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │ 4. Code Contributions: Submitting first PR to master   │
  └────────────────────────────────────────────────────────┘
```

* **Target Hubs:** Focus on engineering college chapters in Andhra Pradesh and Telangana (e.g., JNTU, Osmania University affiliates).
* **Local Hosts:** Appoint Campus Chapter Leads who run monthly open-source meetups.
* **Lab Installations:** Encourage departments to set up DeepDive AI on local intranets, allowing students with limited internet access to run local inference and contribute.

---

## 3. Regional Open-Source Communities

Beyond Swecha, we collaborate with broader open-source communities across India:
* **FOSS United Chapters:** Partnering with regional chapters (e.g., FOSS United Hyderabad, FOSS United Pune, FOSS United Mumbai) to host developer talks and recruit contributors.
* **Python Developer Groups (PyData / PyCon India):** Showcasing our Streamlit frontend, local caching architectures, and ReportLab implementations to regional Python groups to attract experienced developers.
* **Localization Groups:** Partnering with Indic NLP groups to source translations for regional languages beyond Telugu, Hindi, and Marathi.

---

## 4. Hackathon Participation Strategy

Hackathons provide high-intensity onboarding environments:
* **Challenge Track:** Sponsor a "Local AI & Multilingual Tooling" track at regional hackathons.
* **Starter Kits:** Provide hackathon teams with ready-to-run Docker configurations and local model blueprints.
* **Fast-track Reviews:** Dedicate maintainers to review hackathon PR submissions in real-time, ensuring successful projects are integrated into the main branch.
