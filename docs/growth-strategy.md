# DeepDive AI — User Growth & Outreach Strategy

This document outlines the strategy for driving user acquisition, engagement, and retention for DeepDive AI. It serves as a blueprint for expanding the project's user base over an 8-week horizon and beyond.

---

## 1. Project Overview & Value Proposition

**DeepDive AI** is a Streamlit-based web application that leverages Google Gemini and local Ollama inference to generate high-fidelity, structured research reports (complete with Executive Summaries, Key Insights, Statistics, and Sources). Users can download these outputs as publication-ready PDFs and PowerPoint presentation decks. It also features a secure, local-first Document Q&A (RAG) feature to analyze local PDFs privately.

### Key Strengths:
* **Dual Backend:** Run cloud inference via Gemini (BYOK) or 100% offline via Ollama.
* **Multilingual UI & Output:** Full support for English, Hindi, Marathi, and Telugu.
* **Instant Export:** One-click downloads for PDF and PPTX decks.

---

## 2. Target User Personas

We categorize our primary user base into four target personas:

### A. Students & Academic Researchers
* **Needs:** Fast synthesis of new domains, structured reference gathering, and quick slide/PDF generation for presentations or literature reviews.
* **Pain Point:** Spending hours manually formatting references and creating summary slides.
* **Benefit:** Instant draft generation and pre-formatted downloads.

### B. Knowledge Workers & Business Analysts
* **Needs:** Briefings on market trends, tech disruptions, or industry terminology.
* **Pain Point:** Information overload and paid API boundaries.
* **Benefit:** Local-first, private PDF document analysis (RAG) ensures sensitive business documents never leave their machine.

### C. Local AI & Open-Source Enthusiasts
* **Needs:** Running LLMs locally without sharing data with proprietary clouds, testing open weights models (Mistral, LLaMA 3, Gemma, Qwen).
* **Pain Point:** Lack of simple user interfaces for local models.
* **Benefit:** Complete out-of-the-box local Streamlit UI that coordinates with Ollama.

### D. Digital Autonomy & Language Advocates
* **Needs:** High-quality research reports and UI interactions in regional Indian languages (Hindi, Marathi, Telugu).
* **Pain Point:** Most advanced research generators are English-only.
* **Benefit:** Native translation prompts and localized UI interfaces.

## 3. 8-Week User Acquisition Plan

Our structured outreach plan focuses on iterative channels to achieve a self-sustaining user growth loop.

```
┌────────────────────────────────────────────────────────┐
│  W1-W2: Foundations & Digital Launch                   │
├────────────────────────────────────────────────────────┤
│  W3-W4: Academic & University Onboarding               │
├────────────────────────────────────────────────────────┤
│  W5-W6: Community Workshops & Regional Demonstrations  │
├────────────────────────────────────────────────────────┤
│  W7-W8: Optimization, Expansion & Feedback Integration │
└────────────────────────────────────────────────────────┘
```

### Weeks 1 - 2: Digital Launch & Foundation
* **Goal:** Establish digital visibility and self-hostable packages.
* **Activities:**
  * Publish the application on Streamlit Community Cloud and Render.
  * Share the project on public directories (Hacker News, Product Hunt, Reddit communities like `r/selfhosted`, `r/LocalLLaMA`, and `r/streamlit`).
  * Publish setup guides and troubleshooting walkthroughs on Dev.to and Medium.

### Weeks 3 - 4: Academic & University Onboarding
* **Goal:** Direct onboarding of student and academic user personas.
* **Activities:**
  * Engage with Swecha-affiliated college chapters and academic computer science departments.
  * Distribute digital flyers outlining how DeepDive AI saves time during seminar papers and project planning.
  * Run virtual office hours to assist students in setting up Ollama on their personal laptops.

### Weeks 5 - 6: Community Workshops & Local Demos
* **Goal:** Physical and virtual workshops demonstrating the value of offline AI.
* **Activities:**
  * Host interactive workshops demonstrating "Private Document Analysis: Chatting with your PDFs offline using Ollama."
  * Conduct hands-on labs in regional languages (Hindi, Marathi, Telugu) to demonstrate multilingual generation.
  * Collect user feedback in the community JSON database via the sidebar form.

### Weeks 7 - 8: Scale & Partner Integration
* **Goal:** Secure long-term partnerships and implement feedback loops.
* **Activities:**
  * Partner with local non-profits, translation groups, and open-source networks to integrate regional language corpora.
  * Release a "Community Showcase" highlighting research reports generated by users.
  * Integrate feedback from `data/reviews.json` to optimize features and release v1.1.0.

## 4. Marketing & Outreach Channels

### Social Media Plan
* **Twitter/X & Mastodon:** Weekly threads showing before-and-after results (e.g., inputting a raw topic and generating a 7-slide PowerPoint deck in 15 seconds). Tag local AI groups and open-source foundations.
* **LinkedIn:** Focus on the RAG (Document Q&A) security aspect. Write posts explaining how local embedding caches prevent intellectual property leaks.
* **YouTube/YouTube Shorts:** 60-second tutorial videos:
  * "How to run LLaMA 3 locally for research reports in 3 steps."
  * "Generating Hindi/Telugu PDFs offline with Noto Sans fonts."

### Campus Ambassador Strategy
* **Selection:** Appoint student ambassadors in engineering colleges.
* **Mandate:** Host one "DeepDive AI Install Fest" per semester. Assist classmates in installing Python, Streamlit, and pulling Ollama models (`llama3`, `mistral`).
* **Incentives:** Swag (Swecha/DeepDive stickers), certificates of open-source contribution, and mentorship opportunities.

## 5. User Retention Strategy

Acquiring users is only half the battle; keeping them engaged requires value-centric workflows:
1. **Embedding Cache Performance:** The local SQLite cache ensures that repeated queries for Document Q&A load instantly, offering desktop-like speed.
2. **BYOK Flex:** Users can use free Google Gemini keys when they have internet, and fallback seamlessly to Ollama when offline.
3. **No-Friction Updates:** Provide a simple update guide in the app sidebar so users can pull git changes without losing their localized embedding databases.
4. **Prompt Tuning Options:** Allow users to adjust prompt parameters (temperature, token limits) to control the depth of their reports.
